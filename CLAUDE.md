## FEED — Dev/AIニュースダイジェスト
- GitHub Pages配信 (refined1975-ship-it/feed)
- 毎日 7:30 JST LaunchAgent (`~/.local/bin/feed-daily.sh`) → `claude -p` でニュース+論文蒸留+push
- ニュース: 7日保持、論文: 永続蓄積（土日スキップ）
- 3タブ構成: ニュース / 論文 / 記事

## コンテンツ方針
- AI活用Tips (プロンプトの書き方、使い方) を最優先
- AI/認知科学/発達心理学の交差点を混ぜる
- バランス目安: 技術5-6 + AI Tips/認知科学2-3 + その他2-3

## ポンコツエンジニアリング（記事タブ）
- 善悪を判定しない。メカニズムを記述する
- 寄り添い・慰め・他責・自責を入れない。バグレポートのように書く
- 心理学ではなく工学の語彙で記述する
- CS/AIの知識を掛け合わせて、当事者にも新鮮な角度を提供する
- 対策がクリシェ（タスク分割しろ、自分を許せ等）になるなら書かない

## claude -p 注意点
- `exec >> "$LOG" 2>&1` を使わない — stdoutリダイレクトがstdinを汚染する
- `--permission-mode bypassPermissions` を付ける
- プロンプトはtmpファイルに書いてから `< "$PROMPT_FILE"` で渡す
- スクリプトを `&` 付きで呼ばない — バックグラウンドのstdinは /dev/null になる

## ワークフロー
- フロントエンド変更はpush前にローカルで動作確認する
- UIボタンに絵文字を使わない。SVGアイコンを使う
